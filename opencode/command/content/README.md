# Content Commands

Commands for content creation, editing, documentation, and technical writing.

## Commands

### [critique](critique.md)
Content review and improvement suggestions.

### [document](document.md)
Technical documentation creation and structuring.

### [edit](edit.md)
Content editing and refinement for clarity and accuracy.

### [polish](polish.md)
Content polishing and finalization for publication.

## Usage

Commands in this category focus on:
- **Technical Writing**: Clear, accurate technical content
- **Documentation**: Structured documentation systems
- **Content Quality**: Editorial standards and consistency
- **User Experience**: User-focused documentation

## Common Scenarios

- Creating API documentation
- Writing technical guides and tutorials
- Reviewing and improving existing content
- Structuring documentation systems
- Polishing content for publication

## Related Commands

- `/analyze` - Code analysis and documentation needs
- `/review` - Code review with documentation considerations
- `/explain` - Code explanation and documentation
