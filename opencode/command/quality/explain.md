Provide clear, comprehensive explanation of code with expert context.

Explanation Routing:
- Architecture patterns → Master Architect
- Frontend components → Frontend Engineer
- Backend services → Backend Architect
- Database queries/schema → Database Architect
- Security implementations → Security Auditor
- Performance optimizations → Performance Analyst
- Test implementations → Test Automation Engineer
- Mobile code → Mobile Engineer
- Accessibility features → Accessibility Engineer
- CI/CD configurations → Deployment Engineer

Explanation Includes:

**High-Level Overview**
- Purpose and responsibility
- How it fits in the larger system
- Key design decisions and rationale

**Code Walkthrough**
- Line-by-line or section-by-section explanation
- Important patterns and techniques used
- Why specific approaches were chosen
- Trade-offs and considerations

**Technical Details**
- Algorithms and data structures
- Performance characteristics
- Error handling approach
- Edge cases and boundary conditions
- Dependencies and integrations

**Context & Relationships**
- Related components and files
- Data flow and state management
- API contracts and interfaces
- Configuration and environment variables

**Best Practices Applied**
- Design patterns used
- SOLID principles implementation
- Security considerations
- Performance optimizations
- Accessibility features

**Potential Improvements**
- Known limitations
- Future enhancement opportunities
- Alternative approaches
- Refactoring suggestions

Explanation Style:
- Start with high-level overview
- Progress to implementation details
- Use clear, jargon-free language where possible
- Provide examples and analogies
- Reference relevant documentation
- Highlight non-obvious or clever solutions

Perfect for: Onboarding, code reviews, documentation, learning, debugging complex code.
